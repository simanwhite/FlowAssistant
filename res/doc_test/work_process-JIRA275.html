2016-8-22

pg_blk被拿进了hier.pm. 新的错误又出现了, vcs_compile报错, hierarchy结构不对, 找不到
aggr这个inst. 原因是hier.txt就没有在最上面一层加tb这一层, raven1x是tb.athub.aggr, 而
navi10因为用了rhea_drop, 所以hier.pm里面是athub.aggr, 少了tb.
tb是testbench的意思,



为了让Lisa的design的top层设为athub, 修改了以下东西:
    project.rgb.make里的pg_top
    config.dj里的power_top
    top.upf里的东西
把release_test-jira275下面的dump_hier.tmpl改了, 准备直接fuck the result.


Joe的回复:
The parser starts from the design module rather than testbench, so you have to p
repend the tb as top module and replace the design top (athub) with the design t
op instance (athub, though the same here), in your post-process step before you
proceed to use the actual hierarchy path with regard to the tb level.

所以rhea_parser总是从design的角度看, 不会拿testbench的层级. 所以加tb的处理只能在
powerflow_pkg里面做. 做的方式, 如Joe, 应该是这样的:
  1. 判断是不是需要tb层级
  2. hierarchy前面加tb作为top module
  3. 将design top替换成design top instance
