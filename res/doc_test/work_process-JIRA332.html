2016-11-8

This optimization is implemented in powerflow_pkg/rls_test_jira332
I've tried it on raven2x uvd_uvde_t.upf generation and vsi check. It'll cut time cost from 12mins to 7mins with vsi_check running in background.
One risk:
if $nlp_iso is set, then genTileSimUpf.csh will use vsi_rtdb/reports/xover_signal.lst, which is generated by the last vsi_shell command, for isolation policy creation.
I'm adding protection for this risk.
Code related:
if ($nlp_iso) then
if ($vc_lp_check) then
echo "Running vc-lp isolation policy creation for nlp simaultion"
$ANCHOR_powerflow/src/bin/nlp_isolation.pl -xover_file vcst_rtdb/reports/xover_signal.lst -upf ${tile}.upf
else
echo "Running isolation policy creation for nlp simaultion"
$ANCHOR_powerflow/src/bin/nlp_isolation.pl -xover_file vsi_rtdb/reports/xover_signal.lst -upf ${tile}.upf
endif
endif
